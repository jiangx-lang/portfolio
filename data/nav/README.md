# 基金 NAV 历史数据（GitHub Raw）

Streamlit 应用从 GitHub Raw 读取本目录下的 CSV，用于展示基金净值走势与区间/年度收益。  
**URL 示例**：`https://raw.githubusercontent.com/jiangx-lang/portfolio/master/data/nav/`

---

## 工作流（爬虫跑完之后）

```bash
# 1. 把 CSV 放到本地仓库的 data/nav/ 文件夹
# 文件名必须和 MRF_POOL 的 key 完全一致，例如：
data/nav/摩根亚洲总收益.csv
data/nav/摩根国际债.csv
data/nav/东方汇理香港组合-灵活配置均衡.csv
# ... 见下方「文件名清单」

# 2. 提交并推送
git add data/nav/
git commit -m "update nav 2025-03"
git push
```

推送后 Streamlit 会按缓存 TTL（1 小时）自动读到新数据，**无需重启应用**。

---

## CSV 格式要求（给爬虫 / 导出脚本用）

- **表头（仅两列）**：`csvdate,nav`
- **csvdate**：日期，格式 `YYYY-MM-DD`
- **nav**：单位净值，浮点数（如 `1.0000`、`1.0023`）
- **编码**：**UTF-8**（防止中文路径/文件名在 GitHub 上读取出错）
- **行**：按日期升序排列即可（应用内会再排序）

示例：

```csv
csvdate,nav
2020-01-02,1.0000
2020-01-03,1.0023
2020-01-06,1.0018
```

## 给 Cursor 写「导出脚本」时的提示词（复制即用）

> 请写一个脚本，把爬虫得到的基金净值数据导出成 CSV，供 Streamlit 从 GitHub Raw 读取。
>
> **要求：**
> 1. 输出目录：`data/nav/`（或通过参数指定）。
> 2. 每个基金一个文件，**文件名 = 基金名称（与 MRF_POOL 的 key 完全一致）**，扩展名 `.csv`。例如：`摩根亚洲总收益.csv`、`东方汇理香港组合-灵活配置均衡.csv`。
> 3. CSV 格式：仅两列，表头为 `csvdate,nav`。`csvdate` 为 `YYYY-MM-DD`，`nav` 为单位净值浮点数。按日期升序。
> 4. 文件用 **UTF-8 编码**保存。
>
> 若爬虫输出是 DataFrame，则列名映射为：日期列 → `csvdate`，净值列 → `nav`，再按上述格式写出即可。
